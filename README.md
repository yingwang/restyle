# restyle

`restyle` 是一个改中文的写作 skill，用来处理那些语法没有明显错误、意思也能看懂，可读起来仍然像机器整理出来的文字。它会把过早出现的判断放回事情本身，把抽象说法改成具体的人、动作、场景和语气，让每一句都能顺着前面的内容往下说。读者读到后面时，应该能看到事情怎样发生、人物怎样动作、判断为什么会出现，而不是突然看到一句已经替他总结好的话。

它默认保留原文自己的身份、用途和语气。如果原文在写人，改写时就要让人的动作、说话方式和彼此关系更清楚；如果原文在写事，就要把时间顺序、前后原因和中间的转折讲明白；如果原文在说明一个东西，就要先讲清对象、背景和作用，再整理句子的长短和语气。改写时不追求压缩字数，原文该展开的地方要展开，该补背景的地方要补背景，该说明两句话之间关系的地方也要说明清楚，最后留下来的应该是一段内容完整、读起来顺的中文。

很多像机器写的中文，问题出在句子的组织方式上。抽象名词喜欢站在句首，简单动作被写成“进行”“实现”“提供”，一句话靠“通过……从而……”这类结构拖着走，意思看似完整，读起来却不像平常中文。`restyle` 会把这种句子换回中文更常见的顺序，让人知道谁在做什么，事情怎样发生，情绪从哪里出现，下一句为什么接得上。

默认模式只按原文自己的方向改。只有你明确写了 `--like 张爱玲`、`--like 汪曾祺` 这样的参数，它才会参考对应作家或作品的写法，比如句子通常怎样展开，叙述者会把注意力放在哪里，什么时候说明，什么时候停下来让事情自己呈现。参考归参考，内容仍然以原文为准，原文没有的材料、情绪和场景都不主动添加。提到汪曾祺时，主要是提醒模型别端着，把人、物和动作写清楚，把句子按事情发生的顺序写下去，不能把所有稿子都改成同一种口吻。

## 安装

```bash
ln -s "$PWD" ~/.claude/skills/restyle
```

也可以直接把这个目录拷到 `~/.claude/skills/` 下面。

## 使用

```
/restyle <贴进来的文字 | 文件路径> [--like 作家/作品] [--clean]
```

`--like 张爱玲` 会在保留原文内容的前提下，参考一点指定作家或作品的写法。`--clean` 只输出改好的正文，不附改动说明。更完整的规则、处理顺序和例子写在 `SKILL.md` 里，这个 skill 的核心是保留信息，把话讲清楚，让段落里的内容前后连起来。
