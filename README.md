# restyle

`restyle` 是一个改中文的写作 skill，用来处理那些没有明显语病、意思也能看懂，但读起来仍然像机器整理出来的文字。它不会把原文压成摘要，也不会只替换几个词。它会先判断原文要完成什么任务，再把提前出现的判断放回具体材料里，让人物的动作、事情的先后、场景里的信息和语气之间形成关系。

不同稿子的处理重点不一样。一篇写人的稿子，如果只剩几个判断，读者就看不见人，因此改写要把动作、说话方式和人物关系补清楚。写事的稿子更看重时间顺序、前后原因和中间的转折。说明性的文字要先让读者知道对象是什么、背景是什么、它起什么作用，然后再整理句子的长短和语气。改写后的文字应该保留原文的信息和用途，读起来像重新写过的一段中文，而不是一组被重新排列过的短句。

很多机器味中文的问题出在句子的组织方式上。原文可能让抽象名词站在句首，把简单动作写成“进行”“实现”“提供”，或者用“通过……从而……”把一句话拖得很长，却没有让读者看清谁在做什么。`restyle` 会把这些句子换回中文更常见的顺序，让动作、原因和判断之间的关系变得清楚。

默认模式只按原文自己的方向改。只有明确写了 `--like 张爱玲`、`--like 汪曾祺` 这样的参数，它才会参考对应作家或作品的写法。参考时仍然以原文内容为准，原文没有的材料、情绪和场景都不主动添加。

## 安装

```bash
ln -s "$PWD" ~/.claude/skills/restyle
```

也可以直接把这个目录拷到 `~/.claude/skills/` 下面。

## 使用

```
/restyle <贴进来的文字 | 文件路径> [--like 作家/作品] [--clean]
```

`--like 张爱玲` 会在保留原文内容的前提下，参考一点指定作家或作品的写法。`--clean` 只输出改好的正文，不附改动说明。更完整的规则、处理顺序和例子写在 `SKILL.md` 里。
