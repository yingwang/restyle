# restyle

`restyle` 是一个用来改中文的写作 skill。它处理的不是错别字和简单润色，而是那种“看起来没错，读起来却像机器写的”中文：判断太满，句子太顺滑，意思被提前概括好，段落里没有真实的动作和承接。它会把原文重新写得自然、具体、连贯、舒展一些，让话像是从材料里长出来，而不是从几个抽象标签里推出来。

这个 skill 不做摘要，也不默认压缩字数。原文有内容，就把内容写清楚；原文该展开，就让句子多走几步。它会尽量保留原文自己的身份、用途和语气，不把所有文字都改成同一种散文腔。写人的时候，让人物的动作和话语站住；写事的时候，把前后关系交代顺；写说明文字的时候，先把对象讲明白，再处理节奏和语气。

很多机器味中文的问题，不在于用了几个明显的套话，而在于句子的骨架已经被英文和报告腔带走了。它会习惯用抽象名词当主语，用“通过……从而……”把简单动作架起来，用“具有”“提供”“实现”“带来”这类词替代真正的动词。`restyle` 的目标是把这些句子拆回中文自己的顺序：谁在做什么，事情怎么发生，上一句怎样接到下一句，哪里需要停一下，哪里可以继续往下说。

如果不给参数，它默认按原文自己的方向改。`--like` 只在你明确指定某个作家或作品时才生效；即使用了它，也只是借节奏、观察角度和句法倾向，不会把某个作家的常见意象当成素材包硬塞进原文。汪曾祺可以作为一种参照：平实、准确、从容，但不是要把所有稿子都写成汪曾祺。

## 安装

```bash
ln -s "$PWD" ~/.claude/skills/restyle
```

也可以直接把这个目录拷到 `~/.claude/skills/` 下面。

## 使用

```
/restyle <贴进来的文字 | 文件路径> [--like 作家/作品] [--clean]
```

- `--like 张爱玲`：在保留原文内容的前提下，额外借一点指定作家或作品的高层风格特征。
- `--clean`：只输出改好的正文，不附改动说明。

更完整的规则、处理顺序和例子写在 `SKILL.md` 里。这个 skill 的重点不是把文字改短，而是把话说顺、说透，让段落有来处，也有气口。
