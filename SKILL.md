# restyle

把原文改成读者能顺着内容读下去的中文，并保留原文里的信息、用途和语气。改写时不要只把句子改短，也不要把几句短话用逗号拼成一句长话。真正要做的是重新安排句子内部和段落之间的关系，让读者看得出谁在做什么，事情为什么这样发展，某个判断又是从哪些动作、场景或语气里来的。

很多像机器写的中文，会在事情还没有展开时就先给出结论。读者还没有看见人物怎样动作，也没有看见事情怎样发生，原文就已经把情绪写成了几个词，或者把一段经历概括成一句看似完整的话。处理这类稿子时，要把结论放回原文已经给出的材料里，让判断从人、事、动作和前后变化中出现。

机器写的文字还常见两种问题。英文式的组织方式会让抽象名词站在句首，让句子靠“通过”“实现”“具有”“提供”这类词往下拖。清单式写法则把动作一项项列出来，却没有交代动作之间的关系。处理这些问题时，不要只替换几个词，也不要为了让篇幅变短删掉信息，而要重新组织主语、动作、原因、转折和补充说明。

## 先判断原文要做什么

改之前先读原文，看清它本来在完成什么任务。它可能是在讲一件事，也可能是在写一个人、说明一个产品、整理一段旅行导览、表达一段情绪，或者把某个观点讲得更清楚。文体不同，改写的方向也不同：叙事文字需要过程，说明文字需要层次，导览文字要像站在现场慢慢讲，评论文字要让判断和依据同时站住。

判断清楚以后，再决定怎样改。给别人看的说明要把对象、背景、动作和结果讲明白，日记或私人文字要保留原来的迟疑、停顿和亲近感，导览词则要让观众知道眼前在看什么、它为什么值得看，以及它和前后的空间怎样接上。文字可以变顺，原文的身份不能被换掉。

## 只用原文已有的材料

改写只能从原文已有的材料里生长。人物、地点、时间、物件、声音、动作和已经出现的判断，都可以重新组织，但是原文没有给出的事实不能为了画面感自己补上。当材料足够时，要把它们之间的关系写开。读者需要知道谁先做了什么，另一件事为什么接上来，情绪从哪个动作里露出来，哪一句需要多给一点背景。

细节要有用。一个细节如果不能帮助读者理解人物、关系、空间或语气，就不该因为“看起来有生活气”而被加进去。粥、咸菜、风、树影、黄昏、旧街道这些东西，只有在原文已经给出，或者确实和原文内容有关时才可以保留和展开。如果它们只是为了制造某种味道，就应该删掉。

材料不够时，写得克制一点，并在“改了什么”里说明缺少哪些细节。抽象情绪不必一概删掉，真实的犹豫、烦躁、难堪、亲近和迟疑可以留下，但它们需要通过动作、说话方式和上下文说明白，不能只靠一个形容词撑住。

## 重新组织短句

少用短句，不等于把句号改成逗号。可以先把几个相关短句放在一起看，但合在一起以后必须重新写。重写时要调整语序，合并重复的主语，补出被省略的原因、条件或转折。那些只是在重复节奏、没有提供新信息的词，也应该删掉。只有这样，长句才会像原本就应该这样写，而不是几个短句临时接在一起。

比如“他进门。他坐下。他沉默。”不能只改成“他进门，坐下，沉默。” 这样的句子虽然变长了，读起来仍然像提纲。更好的改法要回到原文材料里，看他进门后看见了什么，为什么坐下，以及沉默和前面的动作有什么关系。如果原文没有这些信息，就不要假装已经写出了完整过程，而要说明缺少支撑这个反应的细节。

长句可以用，而且很多时候应该用。一个长句是否成立，要看它有没有清楚的主干，补充说明是否围绕主干展开，原因、条件和转折是否放在读者容易理解的位置。标点少并不会自动让句子更好，真正重要的是句子里的关系是否清楚。

## 处理英文式句子

英文式句子常常出在整句话的组织方式上。改的时候先看主语和动词：抽象名词是否站在句首，一个简单动作是否被写成“进行某种处理”“实现某种转变”“提供某种支持”，句子是否绕着“对于……而言”“通过……从而……”“这使得……能够……”“被认为是……”“在……方面具有……”往下写。

中文要靠人、事、动作和语气往下说，少靠抽象名词搭架子。能换成普通动词，就换成普通动词。能把主语说成具体的人或事，就不要让抽象概念站在句首。比如“这使得他能够更好地理解母亲”可以改成“这样一来，他才慢慢懂了母亲”。“通过反复练习，他实现了能力的提升”可以改成“他反复练，手上慢慢熟了”。

改完以后要再读一遍，确认句子已经回到中文自己的顺序里。主语要清楚，动作要清楚，前后要能连起来。

## 处理清单式段落

有些文字只列事项却不写过程，只给结论却不作说明。除非原文就是说明文或论证文，而且确实需要清楚分层，否则少用“首先、其次、最后”“一方面、另一方面”“这体现了”来搭架子，也别只罗列动作点，把段落写成一份分镜表。

如果原文已经像清单，改写不能把它压得更像清单，而要根据原文已有的信息，把必要的背景、动作顺序、原因和转折补清楚。删掉空泛总结时，也要确认具体的人、事、动作、时间、地点没有一起被删掉，因为这些材料才是文章能够读下去的基础。

## 处理临时概括的说法

机器写作很爱把本来应该展开说的意思压成一个小标签，比如“旧 X”“新 X”“前 X”“后 X”“非 X 非 Y”“不 X 也不 Y”“半 X 半 Y”“X 感”“X 性”“X 化”。这些词看着精炼，实际很像临时想出来的标题，正常人平时很少这样写。除非原文已经这么写，或者这个词本来就是常用表达，否则尽量拆开。

同样要小心一两个字的形容词。它们本身未必有问题，但如果一句话本来应该说明人物做了什么、事情为什么变了、某种感受从哪里来，最后只剩“温暖”“治愈”“松弛”“高级”“真实”这类词，读者就看不到这些判断具体指向什么。遇到这种情况，不要急着再找一个更好听的词替换。先回到原文材料里，把动作、关系和前后变化写出来。

把这种标签拆成普通句子，让意思落回事情里。比如“旧关系和新生活”可以写成“过去那段关系还在影响她，可眼前的日子已经往前走了”，“不告别也不重逢”可以写成“他们没有认真告别，后来也没有再见过”，“情绪滞留感”可以写成“那股情绪没有过去，还在她身上停着”。

## 保留原文自己的声音

默认按原文自己的方向改。原文是谁写的，就先尽量保留它的身份、用途和语气。提到汪曾祺时，主要是提醒模型别端着，把人、物和动作写清楚，把句子按事情发生的顺序写下去。不能一看到这个名字，就把稿子改成同一种口吻，也不能硬加吃食、草木、季节这些材料。

真正要抓的是中文本身的顺。朴素说明就按说明写清楚，导游词就像站在现场慢慢讲，私人叙述就保留它原来的亲疏、停顿和含糊。把话写清楚、写足、读起来顺，比“像某个作家”更重要。

## 一个例子

下面这个例子里，前面已经写出了窗缝、锅盖、咸菜、碗和筷子，最后一句又把这些细节压成一句感想。改写时要保留那些具体动作，让读者从动作和场景里看到这顿早饭带来的安定感。

> 清晨外婆给我煮粥。厨房窗子开了一条缝，早上的风从窗缝进来，锅里的热气没有散，锅盖被热气顶得轻轻响。桌上有一碟昨晚切好的咸菜和两只碗。她弯腰从柜子里找筷子，找到了以后在碗边磕了一下，让我先坐，说粥还烫。看着这一幕，我忽然觉得，这样的清晨比任何大道理都更能让人安定。

改后：

> 天刚亮，厨房里已经有声音。窗子开着一条缝，早上的风从那里进来，锅里的热气却没有散，锅盖被顶得轻轻响一下，又落回去。外婆站在灶前看粥，听见我进来，也没有回头，只说先坐，粥还烫。桌上已经摆了两只碗，一碟咸菜是昨晚切好的，细细地堆在盘子里。她弯腰从柜子里找筷子，找到了，拿在手里磕了一下碗边，才把其中一双递给我。

这里删掉了最后那句替读者归纳的感受，但没有把文章压短。窗缝、锅盖、咸菜、碗、筷子和“粥还烫”都保留下来，只是按人在厨房里的顺序慢慢展开。句子不追短，用逗号把动作连起来，读起来像一件事正在发生。

## 使用 `--like`

默认只改原文。用户写了 `--like <作家/作品>`，才额外参考指定对象的写法，比如句子通常怎样展开，叙述者会把注意力放在哪里，会不会直接替读者下判断。参考归参考，内容仍然以原文为准，原文没有的材料、场景和情绪都不主动添加。

使用 `--like` 时，改写正文前面加一小节“参考方式”，说明你参考了哪些具体写法，方便用户判断方向。这里要写得具体，避开“语言优美、情感细腻、富有文学性”这类空话。

手头如果有用户自己定稿的材料，比如 `~/claude/july-2026-travel-book` 或 `~/journal/`，在改用户自己的稿子时可以先扫几段，摸一下它原本的句子长度、转折方式和叙述习惯。重点看句子怎么展开，怎么顺着上一句往下写，别只看用词。

## 输出

默认输出两段：

```
## 改写
<全文>

## 改了什么
- “原片段”改成“新片段”：一句话说清为什么这样改
```

`--like` 模式在最前面加一节：

```
## 参考方式
<说明参考了哪些写法>
```

`--clean` 只输出改写后的正文，不加标题、说明或参考方式。

这个 skill 手动调用，不挂 cron。
