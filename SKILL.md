# restyle

你要做的是把一段中文重新写得像人写的，读起来自然、具体、连贯、舒展，而且信息量要保住。不要把它改成摘要，也不要为了显得干净就把字数压下去。很多稿子的问题不是“太长”，而是话说得没有来处：判断先到了，事情没有铺开；情绪已经被命名了，人还没有出现；段落看上去很完整，里面却只有几个抽象结论。改写时要把这些结论拆回具体材料里，让人物、动作、场景、语气和前后关系把意思托住。

机器写的中文常常有几种毛病。第一，它太会替人总结，总喜欢把一段事收束成一个漂亮判断。第二，它喜欢把中文写成翻译过来的报告句，抽象名词太重，动词太弱，句子靠“通过”“实现”“具有”“提供”往前拖。第三，它容易写成提纲，事情被切成一个个动作点，段落之间没有缓冲和转身。`restyle` 要处理的正是这些问题：不是换几个词，而是重排句子里的动作、主语、气口和承接。

## 先判断原文要做什么

改之前先读原文，不要急着套一种固定文风。原文可能是在讲一件事，可能是在写一个人，也可能是在说明一个产品、整理一段旅行导览、表达一段情绪，或者替某个观点找更清楚的说法。不同文体的自然感不一样。叙事文字要有过程，说明文字要有层次，导览文字要像站在现场慢慢讲，评论文字要把判断和依据放在一起。不要把所有内容都改成散文，也不要把所有段落都改成短句。

先认清它本来的用途，再决定该往哪里改。如果原文是给别人看的说明，就把对象、背景、动作和结果讲清楚；如果原文是日记或私人文字，就保留它的语气和迟疑；如果原文是导览词，就让观众知道自己眼前在看什么、为什么值得看、和前后空间有什么关系。文字可以变顺，但不能换掉原文的身份。

## 从已有材料里长出来

改写只能从原文已有的材料里生长。人物、地点、时间、物件、声音、动作、已经出现的判断，都可以拿来重新组织；原文没有的事实，不要为了“有画面”自己补。材料足够时，就把它们之间的关系写开：谁先做了什么，另一件事为什么接上来，情绪是从哪个动作里露出来的。材料不够时，就写得克制一点，必要时在“改了什么”里说明原文缺少可落地的细节。

具体不是堆名词，详细也不是注水。一个细节必须有用，要能让人更清楚地看见人物、关系、空间或语气。粥、咸菜、风、树影、黄昏、旧街道这些东西，如果原文没有，或者放进去只是为了显得像生活，就不要放。真正好的具体，是从原文已经给出的东西里找到顺序和重量。

## 重写时怎样处理句子

不要追求短句。中文的自然感常常在逗号里，在一句话内部的停顿、转身、补充和承接里。能一口气说完的，就让它一口气说完；该停的时候再停。长句可以用，而且很多时候应该用。只要主干清楚，读者不用回头找主语和谓语，长句会比一串短句更自然，也更像真实的人在慢慢把事情讲明白。

句子要有长短，但长短由气口决定，不由“去 AI 味”决定。不要把段落写成“他起床。他做饭。他看见她。他沉默。”这种动作清单，也不要一句一个镜头地硬切。好的改写应该让上一句自然带出下一句，让事情一点点往前走。读完以后，读者不仅知道发生了什么，还知道它是怎样发生的。

## 去掉翻译腔

翻译腔不只是几个词的问题，更多时候是整句的骨架已经不中文了。改的时候先看句子的主语和动词。是不是抽象名词在当主语？是不是一个很简单的动作，被写成“进行某种处理”“实现某种转变”“提供某种支持”？是不是绕着“对于……而言”“通过……从而……”“这使得……能够……”“被认为是……”“在……方面具有……”往前走？

中文要靠人、事、动作和语气往前走，不要靠一串抽象名词架起来。少用“进行”“作出”“实现”“具有”“提供了”“带来了”“基于”“通过……从而……”“在……过程中”“其”“该”这类报告腔和翻译腔。能换成普通动词，就换成普通动词；能把主语说成具体的人或事，就不要让抽象概念站在句首。

可以这样改：

- “这使得他能够更好地理解母亲”改成“这样一来，他才慢慢懂了母亲”。
- “对于她而言，离开并不是一个容易的选择”改成“她要走，并不容易”。
- “这段经历带来了重要的改变”改成“那段日子以后，她变了不少”。
- “他被认为是一个可靠的人”改成“大家都觉得他靠得住”。
- “通过反复练习，他实现了能力的提升”改成“他反复练，手上慢慢熟了”。

改完以后再读一遍，确认句子不是从英文结构翻过来的，而是像中文本来会说的那样往前走。

## 不要写成提纲

提纲腔的问题，是只有事项，没有过程；只有结论，没有承接。不要用“首先、其次、最后”“一方面、另一方面”“这体现了”来硬搭架子，除非原文就是说明文或论证文，而且这些结构确实能帮读者看清层次。也不要只罗列动作点，让段落变成一份分镜表。

段落要有肉。动作之间要有过渡，物件和人之间要有关系，句子之间要能接住。写完后问自己：这段话是不是只剩“发生了什么”的骨架？有没有写出事情怎样一点点发生？有没有让读者知道前一句为什么接到后一句？如果答案是否定的，就还没改好。

## 不要临时造概括词

机器写作很爱把本来应该展开说的意思压成一个小标签，比如“旧 X”“新 X”“前 X”“后 X”“非 X 非 Y”“不 X 也不 Y”“半 X 半 Y”“X 感”“X 性”“X 化”。这些词有时看着精炼，实际上很像临时想出来的标题，正常人平时不会这样写。除非原文已经这么写，或者这个词本来就是常用表达，否则不要新造。

把这种标签拆成普通句子，让意思落回事情里：

- 不写“旧关系和新生活”，写“过去那段关系还在影响她，可眼前的日子已经往前走了”。
- 不写“不告别也不重逢”，写“他们没有认真告别，后来也没有再见过”。
- 不写“新的自我叙事”，写“她开始换一种方式讲自己的事”。
- 不写“情绪滞留感”，写“那股情绪没有过去，还在她身上停着”。
- 不写“关系的冷处理”，写“两个人都不再主动提，也不再追问”。

可以用已有的普通词、熟语和成语，但不要为了显得凝练就发明概念。拿不准时，宁可多写半句，把话说开。

## 保留原文自己的声音

默认不要模仿任何作家。原文是谁写的，就先尽量保留它自己的身份、用途和语气。汪曾祺可以作为一种参照，但只借他的平、准、从容，不借成固定腔调，不借成短句模板，也不借成吃食、草木、节令的道具。

真正要抓的是中文本身的顺。一个段落如果原本是朴素说明，就不要改成抒情散文；如果原本是导游词，就不要改成论文式介绍；如果原本是私人叙述，就不要把它拔高成通用人生感悟。把话写清楚、写足、写得有气口，比“像某个作家”更重要。

## 一个例子

原文材料已经有了，但最后替读者总结得太满：

> 清晨外婆给我煮粥。厨房窗子开了一条缝，早上的风从窗缝进来，锅里的热气没有散，锅盖被热气顶得轻轻响。桌上有一碟昨晚切好的咸菜和两只碗。她弯腰从柜子里找筷子，找到了以后在碗边磕了一下，让我先坐，说粥还烫。看着这一幕，我深深感受到家的味道，并明白幸福不是惊天动地的事情，而是有人愿意为你早起，把平凡的日子过得温暖。

改后：

> 天刚亮，厨房里已经有声音。窗子开着一条缝，早上的风从那里进来，锅里的热气却没有散，锅盖被顶得轻轻响一下，又落回去。外婆站在灶前看粥，听见我进来，也没有回头，只说先坐，粥还烫。桌上已经摆了两只碗，一碟咸菜是昨晚切好的，细细地堆在盘子里。她弯腰从柜子里找筷子，找到了，拿在手里磕了一下碗边，才把其中一双递给我。

这里删掉了“家的味道”“幸福不是惊天动地”这些已经替读者归纳好的话，但没有把文章压短。窗缝、锅盖、咸菜、碗、筷子和“粥还烫”都保留下来，只是按人在厨房里的顺序慢慢展开。句子不追短，用逗号把动作接起来，读起来像一件事正在发生，而不是像一段作文结尾。

## 使用 `--like`

默认只改原文，不借别人的腔。只有用户写了 `--like <作家/作品>`，才额外参考指定对象的高层特征，比如节奏、语气、观察角度和句法倾向。不要追求表面复刻，不要把某个作家的常见意象当成素材包往原文里塞，也不要为了像某个人而牺牲流畅、详细和自然。

如果使用 `--like`，净稿前面加一小节“风格画像”，说明你抓住了哪几个特征，方便用户判断方向是不是对。风格画像要具体，不要写成“语言优美、情感细腻、富有文学性”这类空话。

手头如果有用户自己定稿的材料，比如 `~/claude/july-2026-travel-book` 或 `~/journal/`，在改用户自己的稿子时可以先扫几段，摸一下它原本的句子长度、转折方式和叙述习惯。重点看句子怎么展开，怎么接住上一句，不是只看用词。

## 几条硬规则

- 不许写成短句大纲。目标是自然、流畅、详细、充分，话要说得开，句子要连得上气。
- 不许为了省字数删掉必要的承接、背景和动作。要去掉的是空话，不是血肉。
- 允许长句，也鼓励顺的长句。长句必须主干清楚，层次分明，逗号用来承接和转弯，不用来硬凑长度。
- 不许翻译腔。少用名词化、被动句、报告腔和英文式结构，让中文靠动词、语气和自然承接往前走。
- 不许提纲腔。不要罗列动作点，不要写成摘要、分镜、会议纪要或作文提纲。
- 不许自造概括词。不要自己造“旧 X”“新 X”“不 X 也不 Y”“X 感”“X 性”这类说法，能写成正常句子就写成句子。
- 不用中文破折号。可以改成逗号、冒号、句号，或者把整句话重写顺。
- 不用 emoji。
- 用全角标点。
- 不许加原文没有的事实和情绪。手头材料不够，就说明哪里缺，不要编。

## 输出

默认输出两段：

```
## 改写
<全文>

## 改了什么
- “原片段”改成“新片段”：一句话说清为什么这样改
```

`--like` 模式在最前面加一节：

```
## 风格画像
<说明借了哪些高层风格特征>
```

`--clean` 只输出改写后的正文，不加标题、说明或风格画像。

这个 skill 手动调用，不挂 cron。
