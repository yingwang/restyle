# restyle

你要把一段中文改成读者能顺着内容读下去的版本，同时保住原文里的信息。很多稿子看上去完整，问题出在判断来得太早，事情还没铺开，情绪已经被概括成几个词；读者还没看到人物怎样动作、事情怎样发生，就先看到一段替他总结好的话。改写时要把这些结论放回原文已经给出的材料里，通过人物、动作、场景、语气和前后关系把意思说清楚。

机器写的中文常常急着替人总结，一段事还没展开，就已经被概括成一句看似完整的结论。它也爱用英文式的报告句，抽象名词重，动词弱，句子靠“通过”“实现”“具有”“提供”往下拖。等到写叙事或说明时，又容易把事情切成一个个动作点，段落之间缺少必要的解释和转折。处理这些问题时，重点放在句子的主语、动作和前后关系上，不能只换几个词，也不能为了让篇幅变短就删掉原文里的信息。

## 先判断原文要做什么

改之前先读原文，看清它本来在做什么。它可能在讲一件事，写一个人，说明一个产品，整理一段旅行导览，表达一段情绪，也可能在替某个观点找更清楚的说法。不同文体需要的写法不一样，叙事文字要有过程，说明文字要有层次，导览文字要像站在现场慢慢讲，评论文字要让判断和依据一起出现。

认清用途以后，再决定语气和结构。给别人看的说明，要把对象、背景、动作和结果讲清楚；日记或私人文字，要保留原来的迟疑、停顿和亲近感；导览词要让观众知道眼前在看什么、它为什么值得看、它和前后的空间怎样接上。文字可以变顺，原文的身份不能被换掉。

## 从已有材料里长出来

改写只能从原文已有的材料里生长。人物、地点、时间、物件、声音、动作、已经出现的判断，都可以重新组织；原文没有给出的事实，不能为了画面感自己补上。材料足够时，就把它们之间的关系写开，比如谁先做了什么，另一件事为什么接上来，情绪从哪个动作里露出来，哪一句需要多给一点背景。

具体要看细节有没有用，详细也要靠材料本身撑住。一个细节要能让人更清楚地看见人物、关系、空间或语气。粥、咸菜、风、树影、黄昏、旧街道这些东西，如果原文没有，或者放进去只是在装生活气，就该留在外面。真正有效的细节，要来自原文已经给出的内容，并且能帮助读者理解人物、场景或判断。

材料不够时，写得克制一点，在“改了什么”里说明缺少可落地的细节。遇到抽象情绪，也不用一概删掉，真实的犹豫、烦躁、难堪、亲近和迟疑可以留下，只要它们能靠动作、语气和上下文说明白。

## 句子和节奏

少用短句，不等于把句号改成逗号。真正要改的是句子之间的关系：前一句为什么引出后一句，后一句是在补充、解释、转折，还是把动作继续往下推。如果只是把“他进门。他坐下。他沉默。”改成“他进门，坐下，沉默。”，句子虽然变长了，写法仍然像提纲。改写时要根据原文已有的信息补出关系，比如他为什么沉默，沉默之前看见了什么，坐下这个动作和后面的反应有什么联系。

长句可以用，而且很多时候应该用，但长句里必须有清楚的主干。读者读完以后，要知道这句话主要说的是谁、做了什么、为什么这样写。补充说明要围绕主干展开，不能把几个互不相干的意思排在一起。一个句子之所以顺，不是因为标点少，而是因为里面的动作、原因、补充和转折都放在合适的位置。

句子要有长短，长短要看这句话本身该怎样停，不能由“去掉机器味”这种外部标准决定。段落不能写成“他起床。他做饭。他看见她。他沉默。”这种动作清单，也不能一句一个镜头地硬切。好的改写应该让上一句自然带出下一句，让事情一点点展开，读完以后，读者不光知道发生了什么，也能知道它是怎样发生的。

## 处理英文式句子

英文式句子常常出在整句话的组织方式上，改的时候先看主语和动词：抽象名词有没有站在句首，一个简单动作有没有被写成“进行某种处理”“实现某种转变”“提供某种支持”，句子有没有绕着“对于……而言”“通过……从而……”“这使得……能够……”“被认为是……”“在……方面具有……”往下写。

中文要靠人、事、动作和语气往下说，少靠抽象名词搭架子。少用“进行”“作出”“实现”“具有”“提供了”“带来了”“基于”“通过……从而……”“在……过程中”“其”“该”这类报告里常见、日常中文里不太自然的写法。能换成普通动词，就换成普通动词。能把主语说成具体的人或事，就别让抽象概念站在句首。

可以这样改：“这使得他能够更好地理解母亲”改成“这样一来，他才慢慢懂了母亲”；“对于她而言，离开并不容易”改成“她要走，并不容易”；“这段经历带来了重要的改变”改成“那段日子以后，她变了不少”；“他被认为是一个可靠的人”改成“大家都觉得他靠得住”；“通过反复练习，他实现了能力的提升”改成“他反复练，手上慢慢熟了”。改完以后再读一遍，确认句子已经回到中文自己的顺序里：主语清楚，动作清楚，前后能连起来。

## 处理只列要点的写法

有些文字只列事项，没有过程；只给结论，没有说明。除非原文就是说明文或论证文，而且确实需要清楚分层，否则少用“首先、其次、最后”“一方面、另一方面”“这体现了”来搭架子，也别只罗列动作点，把段落写成一份分镜表。

段落里要有过程，动作之间要有过渡，物件和人之间要有关系，句子之间要能连起来。写完后读一遍，看它是否只是在列“发生了什么”，看事情怎样一点点发生有没有写清楚，看前一句接到后一句的理由是否清楚。如果只剩条目和结论，就继续补前后的关系和具体过程。

## 处理临时概括的说法

机器写作很爱把本来应该展开说的意思压成一个小标签，比如“旧 X”“新 X”“前 X”“后 X”“非 X 非 Y”“不 X 也不 Y”“半 X 半 Y”“X 感”“X 性”“X 化”。这些词看着精炼，实际很像临时想出来的标题，正常人平时很少这样写。除非原文已经这么写，或者这个词本来就是常用表达，否则尽量拆开。

同样要小心那些一两个字的形容词。它们本身未必有问题，但如果一句话本来应该说明人物做了什么、事情为什么变了、某种感受从哪里来，最后只剩“温暖”“治愈”“松弛”“高级”“真实”这类词，读者就看不到这些判断具体指向什么。遇到这种情况，不要急着再找一个更好听的词替换，而要回到原文材料里，把动作、关系和前后变化写出来。

把这种标签拆成普通句子，让意思落回事情里，比如把“旧关系和新生活”写成“过去那段关系还在影响她，可眼前的日子已经往前走了”，把“不告别也不重逢”写成“他们没有认真告别，后来也没有再见过”，把“新的自我叙事”写成“她开始换一种方式讲自己的事”，把“情绪滞留感”写成“那股情绪没有过去，还在她身上停着”，把“关系的冷处理”写成“两个人都不再主动提，也不再追问”。普通词、熟语和成语可以用，临时发明出来的概念要慎用。拿不准时，多写半句，把话说清楚。

## 保留原文自己的声音

默认按原文自己的方向改。原文是谁写的，就先尽量保留它的身份、用途和语气。提到汪曾祺时，主要是提醒模型别端着，把人、物和动作写清楚，把句子按事情发生的顺序写下去。不能一看到这个名字，就把稿子改成同一种口吻，也不能硬加吃食、草木、季节这些材料。

真正要抓的是中文本身的顺。朴素说明就按说明写清楚，导游词就像站在现场慢慢讲，私人叙述就保留它原来的亲疏、停顿和含糊。把话写清楚、写足、读起来顺，比“像某个作家”更重要。

## 一个例子

下面这个例子里，前面已经写出了窗缝、锅盖、咸菜、碗和筷子，最后一句又把这些细节压成一句感想。改写时要保留那些具体动作，让读者从动作和场景里看到这顿早饭带来的安定感。

> 清晨外婆给我煮粥。厨房窗子开了一条缝，早上的风从窗缝进来，锅里的热气没有散，锅盖被热气顶得轻轻响。桌上有一碟昨晚切好的咸菜和两只碗。她弯腰从柜子里找筷子，找到了以后在碗边磕了一下，让我先坐，说粥还烫。看着这一幕，我忽然觉得，这样的清晨比任何大道理都更能让人安定。

改后：

> 天刚亮，厨房里已经有声音。窗子开着一条缝，早上的风从那里进来，锅里的热气却没有散，锅盖被顶得轻轻响一下，又落回去。外婆站在灶前看粥，听见我进来，也没有回头，只说先坐，粥还烫。桌上已经摆了两只碗，一碟咸菜是昨晚切好的，细细地堆在盘子里。她弯腰从柜子里找筷子，找到了，拿在手里磕了一下碗边，才把其中一双递给我。

这里删掉了最后那句替读者归纳的感受，但没有把文章压短。窗缝、锅盖、咸菜、碗、筷子和“粥还烫”都保留下来，只是按人在厨房里的顺序慢慢展开。句子不追短，用逗号把动作连起来，读起来像一件事正在发生。

## 使用 `--like`

默认只改原文。用户写了 `--like <作家/作品>`，才额外参考指定对象的写法，比如句子通常怎样展开，叙述者会把注意力放在哪里，会不会直接替读者下判断。参考归参考，内容仍然以原文为准，原文没有的材料、场景和情绪都不主动添加。

使用 `--like` 时，改写正文前面加一小节“参考方式”，说明你参考了哪些具体写法，方便用户判断方向。这里要写得具体，避开“语言优美、情感细腻、富有文学性”这类空话。

手头如果有用户自己定稿的材料，比如 `~/claude/july-2026-travel-book` 或 `~/journal/`，在改用户自己的稿子时可以先扫几段，摸一下它原本的句子长度、转折方式和叙述习惯。重点看句子怎么展开，怎么顺着上一句往下写，别只看用词。

## 检查改写质量

改写完成后，先看文章是否只是把要点排在一起。如果原文已经像清单，改写不能把它压得更像清单，而要根据原文已有的信息，把必要的背景、动作顺序、原因和转折补清楚。删掉空泛总结时，也要确认具体的人、事、动作、时间、地点没有一起被删掉，因为这些材料才是文章能够读下去的基础。

长句可以多用，但不能靠替换标点来制造长句。把句号改成逗号、把几条规则挤进同一句，读者仍然看不到这些内容之间的关系。需要合并句子时，先判断这些内容之间有没有真实关系：有因果就把原因写出来，有转折就说明前后差别，有补充就让补充内容紧贴它解释的对象。遇到英文式句子、只列要点的写法和临时造出来的概括说法，要把它们改成普通中文，少用日常说话里不自然的抽象表达，也不要把一串动作点直接排成段落。

像“旧 X”“新 X”“不 X 也不 Y”“X 感”“X 性”这类说法，如果不是原文已有的常用表达，就尽量拆成完整句子。中文破折号也尽量少用，可以改成逗号、冒号、句号，或者把整句话重新写顺。标点用全角，不用 emoji。原文没有的事实和情绪不能添加，手头材料不够时，要说明缺少哪些具体信息，不能自己编。

## 输出

默认输出两段：

```
## 改写
<全文>

## 改了什么
- “原片段”改成“新片段”：一句话说清为什么这样改
```

`--like` 模式在最前面加一节：

```
## 参考方式
<说明参考了哪些写法>
```

`--clean` 只输出改写后的正文，不加标题、说明或参考方式。

这个 skill 手动调用，不挂 cron。
